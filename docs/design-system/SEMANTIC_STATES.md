# Semantic States

Source: `packages/ui/tokens/colors.ts` → `semanticStates`. Rendered via
`<SemanticStatusBadge state="..." />` (`packages/ui/components/Badge.tsx`).

**Rule:** color is never the only signal. Every state below always renders
with an icon and a text label, and exposes an `aria-label` with a fuller
accessible description — a screen-reader user gets the same information as
a sighted user, regardless of color perception.

| State | Label | Icon | Color family | Accessible description |
|---|---|---|---|---|
| `evidenceSufficient` | Evidence sufficient | check-circle | Green | The guideline evidence fully supports this answer. |
| `evidencePartial` | Evidence partial | alert-triangle | Amber | Some, but not all, of this answer is directly supported by cited evidence. |
| `evidenceInsufficient` | Evidence insufficient | ban | Red-orange (distinct from `critical`) | There is not enough approved guideline evidence to answer this safely. Noor has declined to generate a recommendation. **This is a fail-closed safety state, not an error.** |
| `evidenceConflicting` | Evidence conflicting | shuffle | Purple | Approved sources disagree with one another on this point. |
| `verified` | Verified | check-circle | Green | This record has been checked and confirmed accurate. |
| `humanApproved` | Human approved | user-check | Green (distinct icon from `verified`) | A named clinical reviewer approved this content. |
| `aiGenerated` | AI-generated | sparkles | Indigo | This content was generated by Noor's AI pipeline and has not yet received human clinical approval. Always shown, never omitted, never combined visually with `humanApproved`. |
| `warning` | Warning | alert-triangle | Amber | A safety-relevant condition needs your attention. |
| `critical` | Critical | alert-octagon | Red | A critical safety issue requires immediate action. |
| `informational` | Informational | info | Blue | For your information — no action required. |
| `inactive` | Inactive | circle-dashed | Gray | This item is not currently active. |
| `superseded` | Superseded | archive | Blue-gray | A newer version of this guideline replaces this one. |
| `withdrawn` | Withdrawn | x-circle | Dark red | This guideline has been withdrawn and must not be used. Retained for audit only — never shown as retrievable evidence. |
| `processing` | Processing | loader (animated) | Blue | This is currently being processed. |
| `underReview` | Under review | eye | Blue | A clinical reviewer is currently examining this item. |
| `failed` | Failed | alert-octagon | Red | This process did not complete successfully. |

## Why `evidenceInsufficient` isn't just "critical" red

Noor's structured-answer contract (`packages/clinical-schemas`) fails
closed when evidence is insufficient — that's a deliberate, correct safety
behavior, not a system malfunction. Giving it the same visual treatment as
`critical`/`failed` would make a working safety mechanism look like a bug.
It's still a warm/red-adjacent tone (this *is* a stop-and-look state) but
carries its own helper text explaining why, so a clinician isn't left
wondering whether something broke.

## Light/dark

Every state has a light (`fg`/`bg`/`border`) and dark (`fgDark`/`bgDark`/
`borderDark`) value pair, rendered as CSS variables
(`--noor-state-<key>-{fg,bg,border}`) that automatically resolve to the
right pair based on `[data-theme]` or `prefers-color-scheme` — see
`tokensToCssVariables()` in `packages/ui/tokens/index.ts`.
